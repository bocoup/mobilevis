Best Practices for Submitting Examples:

  1. Provide a descriptive name to the original project or application.
  2. Provide the original author for this example. Ideally, use an organization name, like "Propublica" or attribute to a specific creator.
  3. Try to provide a url that will allow users to visit your example live from their browsers or mobile phones.
  4. Provide a description of the original project or application so that people have context for what they are looking at.
  5. Use descriptive tags that will allow others to find your example when looking through collections. Common examples include different visualization types, the type of media (photo, video, sound etc.) if any is used, and areas of potential interest (elections, journalism, health etc.) Don't be shy - the more tags, the better. Try to use tags that already exist.
  6. When uploading images consider taking screenshots of the web version as well. Seeing the differences can be really enlightening! Also, submitting animated gifs is a great idea when you want to show an interesting interaction model or animation.
