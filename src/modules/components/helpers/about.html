What is MobileVis?

Data Visualization is common in every day communication on-line and in print media. As consumers shift to mobile devices for their daily information consumption on the web, we are faced with the challenge of designing new interfaces and interactions that work with these smaller screens. While Responsive Web Design helps alleviate some of these challenges, data visualization brings some additional challenges to the table having to do with specifically displaying and interacting with information.

MobileVis aims to provide both: A collection of examples (and discussion) of data visualization use on mobile devices as well as a collection of patterns that we identify from these examples.

MobileVis is comprised of two parts:

  1. A gallery of examples of data visualization interfaces on mobile devices.
  2. A set of patterns and observed best practices for what works in data visualization on our smaller screens.

  2. Examples are tagged by their submitters. Tags can be visualization types, like Bar Chart or Choropleth or even areas of interest, like Elections. Clicking these tags will reveal other examples that were tagged the same way. When tagging examples of your own, try to reuse existing tags so we can build up these collections!
  3. Examples are submitted by users that are identified with their twitter usernames, like @ireneros. Click the username to see other submissions by this user.
  4. Examples are usually from specific organizations or people, like The Boston Globe. Click the name to see other examples from these organizations. It's a great way to see repeating patterns or templates that may have been used to create content!

Help Make MobileVis Better:

MobileVis is an open source project! You can help us submit or fix bugs, add features and write new design patterns. The web is evolving every day and we don't expect to be able to keep up without your help! Please help us author content so that we can create a lasting place for our community.
