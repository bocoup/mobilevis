What is MobileVis?

Data Visualization is common in every day communication on-line and in print media. As consumers shift to mobile devices for their daily information consumption on the web, we are faced with the challenge of designing new interfaces and interactions that work with these smaller screens. While Responsive Web Design helps alleviate some of these challenges, data visualization brings some additional challenges to the table having to do with specifically displaying and interacting with information.

As we started working on mobile data visualization projects, we had to search for examples across the web to identify common patterns and try to make educated decisions on design patterns that would work based on visual methods and other data-visualization-specific criteria. MobileVis aims to provide both: A collection of examples (and discussion) of data visualization use on mobile devices as well as a collection of patterns that we identify from these examples.

What is MobileVis Actually?

MobileVis is actually comprised of two parts:

  1. A gallery of examples of data visualization interfaces on mobile devices.
  2. A set of patterns and observed best practices for what works in data visualization on our smaller screens.

Submit Examples:

Without users submitting examples, we wouldn't have much to show for! Your contributions would help our community become better, so no example is too small or too trivial and "the more the better" applies here!

You must be logged in to submit an example. We use twitter authentication for our login system. We don't store any of your profile information on our servers - only your twitter handle so that we may associate it with your submissions. We will never tweet on your behalf.

Click on the upload button to add your example once you are logged in.

Best Practices for Submitting Examples:

  1. Provide a descriptive name to the original project or application.
  2. Try to provide a link that will allow users to visit your example live from their browsers or mobile phones.
  3. Provide a description of the original project or application so that people have context for what they are looking at.
  4. Use descriptive tags that will allow others to find your example when looking through collections. Common examples include different visualization types, the type of media (photo, video, sound etc.) if any is used, and areas of potential interest (elections, journalism, health etc.) Don't be shy - the more tags, the better. Try to use tags that already exist.
  5. When uploading images consider taking screenshots of the web version as well. Seeing the differences can be really enlightening!
